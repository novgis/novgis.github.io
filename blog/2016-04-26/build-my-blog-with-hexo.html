程序员要有一个独立博客 —— GitHub Pages 与 Hexo 建站经过

  • hexo教程
  • 建站

摘要：这是一篇记录我搭建独立博客的经过的文章，主要介绍了搭建前的域名注册、GitHub 账号注册等准备工作，搭建过程中 Hexo、GitHub 的部署配置以及参考资料目录。希望对各位看客有所帮助，也给自己新的战场起个开篇，坚持写作与分享。

前言

这篇文章应该算是陈素封老师这篇文章的实践篇，加入了一些我经历过的坑，记录下来，以防换了环境再来部署又要走弯路。这也算是陈老师所说的为什么我们要写自己的博客的原因之一吧。下面以我的博客 www.novgis.com 为例，介绍我在 Windows 7 环境下是如何实践搭建独立博客的过程。

准备工作

域名注册

这部分简单写几句，我的域名是在万网注册购买的，过程很简单，选个自认为有意义的域名直接购买。价格上来说万网的价格不比其它家贵多少，而且万网已经并入阿里云，阿里提供的一些服务都可以免费使用。不得不说，拼服务的时代，阿里走在了前列。购买域名后不能马上通过该域名访问任何网站，原因是还没有做域名解析，在之后的过程中我会介绍如何将域名与你自己的博客绑定。

安装 Node.js

解释下为什么要安装 Node.js，Hexo 是出在一名台湾大学生 tommy351 之手，是一个基于 Node.js 的静态博客程序，Node.js 是 Hexo 的运行环境，非装不可。至于 Hexo 的优点，网上很多文章，我这里就不赘述了。到 Node.js 官网下载相应平台的最新版安装包，一路 Next 安装即可。

安装 Git 并与 GitHub 适配

从 Git 官网下载安装包，同样一路 Next 安装！这里介绍三种打开 Git Bash 的方法：

  • 启动开始菜单中的 Git Bash

  • 鼠标右键启动Git Bash

  • 将 Git 的安装根目录（类似 C:\Program Files\Git）添加到系统环境变量，通过 Win+R 输入 git-bash 启动（我比较常用的）

接下来访问 GitHub 注册一个账号，妥善保存注册邮箱。

通过 ssh key 将本地的 Git 和远程的 GitHub 建立联系： 首先需要检查本地现有的 ssh key，通过上述方式打开 Git Bash，执行命令$ cd ~/.ssh，如果提示 “No such file or directory” 说明是第一次使用 Git，这时我们就要生成新的 ssh key，执行命令：$ ssh-keygen -t rsa -C "youremail@youremail.com"
