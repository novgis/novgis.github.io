程序员要有一个独立博客 —— GitHub Pages 与 Hexo 建站经过

  • 建站

摘要：这是一篇记录我搭建独立博客过程的文章，主要介绍了搭建前的域名注册、GitHub 账号注册等准备工作，搭建过程中 Hexo、GitHub 的部署配置以及参考资料目录。希望对各位看客有所帮助，也给自己新的战场起个开篇，坚持写作与分享。

前言

这篇文章应该算是陈素封老师这篇文章的实践篇，加入了一些我经历过的坑，记录下来，以防换了环境再来部署又要走弯路。下面以我的博客 www.novgis.com 为例，介绍我在 Windows 7 环境下是如何实践搭建独立博客的过程。

准备工作

域名注册

这部分简单写几句，我的域名是在万网注册购买的，过程很简单，选个自认为有意义的域名直接购买。价格上来说万网的价格不比其它家贵多少，而且万网已经并入阿里云，阿里提供的一些服务都可以免费使用。不得不说，拼服务的时代，阿里走在了前列。购买域名后不能马上通过该域名访问任何网站，原因是还没有做域名解析，在之后的过程中我会介绍如何将域名与你自己的博客绑定。

安装 Node.js

去 Node.js 官网下载相应平台的最新版安装包，一路 Next 安装即可。

安装 Git 并与 GitHub 适配

从 Git 官网下载安装包，同样一路 Next 安装！这里介绍三种打开 Git Bash 的方法：

  • 启动开始菜单中的 Git Bash

  • 鼠标右键启动Git Bash

  • 将 Git 的安装根目录（类似 C:\Program Files\Git）添加到系统环境变量，通过 Win+R 输入 git-bash 启动（我比较常用的）

接下来访问 GitHub 注册一个账号，妥善保存注册邮箱。

通过 ssh key 将本地的 Git 和远程的 GitHub 建立联系：

首先需要检查本地现有的 ssh key，通过上述方式打开 Git Bash，执行命令：$ cd ~/.ssh

如果提示 “No such file or directory” 说明是第一次使用 Git，这时我们就要生成新的 ssh key，执行命令：$ ssh-keygen -t rsa -C "youremail@youremail.com"

剩下最后一步，将刚生成的 ssh key 添加到 GitHub 中：

  1. 用文本打开本地C:\Documents and Settings\Administrator.ssh\id_rsa.pub文件。此文件里面内容为刚才生成的公钥。如果看不到这个文件，你需要设置显示隐藏文件。全选复制文件内容。

搭建过程

所有的准备工作都做好后，就可以着手搭建博客了。过程会可能会遇到各种各样的问题，在最后我给出了我在搭建过程中踩过的坑，以供参考。

在 GitHub 中建立仓库

登录 GitHub 首页，点击页面右上角「New repository」新建仓库，填写仓库信息：

  • Repository name: novgis.github.io

  • Description (optional): novgis’s blog

这里一定要注意：Github Pages 的 Repository 名字是特定的，比如我 Github 账号是 novgis，那么我 Github Pages Repository 名字就是 novgis.github.io，这是规定的，必须这样写。

点击「Create repository」完成创建。

这个时候其实再进行一些简单的初始化操作即访问我们自己的博客了（由于 novgis 账户已经正式启用，我将使用别的 github 账号来截图作为参考），进入刚新建的仓库页面，点击「Settings」，把页面往下翻会看到「GitHub Pages」这一栏

点击「Launch automatic page generator」按钮进入页面模板设置页面「New your site」，填写一些网站基本信息后点击「Continue to layouts」进入网站模板选择页面，选择你看中的模板点「Publish page」，至此便可以通过 http://time2goo.github.io 链接来访问博客了。

以这种方式建站后，我们对博客的维护可以通过对该仓库中静态页面的修改来一步步完善，新增、修改、删除等等，路漫漫其修远兮…… 所有的操作都要手动完成，作为一个有逼格的程序员自然会想法子让事情变得简单，于是 Hexo 便应运而生。

Hexo 的飒爽英姿

Hexo 出自一名台湾大学生 tommy351 之手，是一个基于 Node.js 的静态博客程序。至于 Hexo 的优点，网上很多文章，我这里就不赘述了。下面来说说怎样通过 Hexo 让写博客变得简单。

安装 Hexo

打开 Git Bash，执行如下命令：$ npm install -g hexo

可能要稍微等会，我安装的时候因为没有进度显示，感觉像是卡住了，后来我 ctrl+c 暂停然后再执行一次，安装过程就快很多。

Hexo 初始化

安装完成后，要执行 init 命令将 Hexo 初始化到你指定的目录：$ hexo init <folder>

也可以先在本地新建个文件夹，然后在文件夹上单击右键打开 Git Bash 亦或直接打开 Git Bash 然后用 $ cd <folder> 命令切换到你指定的文件夹下执行 $ hexo init （此处就不需要加目录了）

独立域名与 GitHub Pages 博客的绑定

在准备工作中，我们申请的独立域名在这里就有用处了。默认 GitHub Pages 服务为我们生成了形如 novgis.github.io 这样的域名，通过该域名是可以访问自己的博客的，但是为了进一步提高我们的逼格，访问形如 www.novgis.com 这样的域名是不是觉得更好呢！那么我们就来说说怎样将独立域名和GitHub Pages 博客进行绑定。

提交 CNAME 文件
