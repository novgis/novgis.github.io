上篇我们对 JSP 的运行原理及其最基本的语法做了简单总结，这篇我们来学习 JSP 中和属性有关的方法，结合请求的转发和重定向，看看能发挥出怎样的威力！

JSP 中和属性(Attribute)相关的方法

前面我们介绍了 JSP 中包含 9 个隐式对象，其中有这么 4 个对象我们称之为域对象，pageContext、request、session、application，这些对象中都有以下和属性相关的方法：

Object getAttribute(String name);
void setAttribute(String name, Object o);
Enumeration getAttributeNames();
void removeAttribute(String name);

\javaweb\attr_1.jsp

\javaweb\attr_2.jsp

TestServlet.java

pageContext (PageContext)

属性的作用范围仅限于当前 JSP 页面。

request (HttpServletRequest)

属性的作用范围仅限于同一个请求。

session (HttpSession)

属性的作用范围限于一次会话（浏览器打开直到关闭称之为一次会话，前提是在此期间session不失效）

application (ServletContext)

属性的作用范围限于当前 WEB 应用，是范围最大的属性作用域，只要在一处设置属性，在其他各处的 JSP 或 Servlet 中都可以获取属性值。

请求转发与重定向 (重点)

从以上的分析来看，request 似乎没比 pageContext 强多少，那有没有可能让 request 实现跨页面作用域呢？而且有这些与属性相关的方法也并不能做些啥！但是跟请求的转发和重定向结合起来的话将威力无穷！敬请欣赏！
