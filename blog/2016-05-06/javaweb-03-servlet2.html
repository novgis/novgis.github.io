Http 协议简介

Http 协议用于定义 Web 浏览器与 Web 服务器之间交换数据的过程以及数据本身的格式。是一种短暂的无状态的协议，每次连接只处理一个请求和响应。浏览器到 WEB 服务器之间的所有通讯都是完全独立分开的请求和响应对。Http 协议的会话方式如下图：

Http 请求消息的结构

基本结构：一个请求行，若干消息头、以及实体内容，其中的一些消息头和实体内容都是可选的，消息头和实体内容之间要用空行隔开。

Http 响应消息的结构

基本结构：一个状态行、若干消息头、以及实体内容，其中的一些消息头和实体内容都是可选的，消息头和实体内容之间要用空行隔开。

GET 请求和 POST 请求

GET 请求：

  1. 在浏览器地址栏中输入某个 URL 地址或单击网页上某个超链接时，浏览器发出的 HTTP 请求消息的请求方式是 GET 请求
  2. 如果网页中的 <from> 表单元素的 method 属性设置为 ‘GET’，浏览器提交表单时生成的 HTTP 请求是 GET 请求
  3. 使用 GET 请求方式给 WEB 服务器传递参数的格式：在链接后添加键值对
  1. 使用 GET 请求方式传递的数据量一般限制在 1KB 以下

POST 请求：

  1. POST 请求方式主要用于向 WEB 服务器程序提交 FORM 表单中的数据，即 form 表单的 method 属性设置为 POST
  2. POST 方式将各个表单字段元素及其数据作为 HTTP 消息的实体内容发送给 WEB 服务器 ，传送的数据量要比使用 GET 方式大得多

HttpServlet 的实现

抽象类，继承自 GenericServlet，针对 HTTP 协议定制的 Servlet，实际开发中直接根据 HTTP 的请求方式针对性的覆写 doXxx() 方法即可，且可以直接使用 HttpServletRequest 和 HttpServletResponse 而不用再去强转。另外 HttpServlet 默认处理了所有类型的请求（doGet、doPost等方法都有自己的实现），即使在我们自己的 Servlet 中没有处理接收的请求，服务器也默认处理了
