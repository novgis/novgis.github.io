(4) JSP（上）

  • Java Web
programming

JSP(Java Server Page，Java 服务器端网页) 是简化 Servlet 编写的一种技术，它将 Java 代码和 HTML 语句混合在同一个文件中编写，只对网页中的要动态产生的内容采用 Java 代码来编写，而对固定不变的静态内容采用普通静态 HTML 页面的方式编写。

JSP HelloWorld

JSP 页面是由 HTML 语句和嵌套在其中的 Java 代码组成的一个普通文本文件，扩展名为 .jsp。在 JSP 页面中编写的 Java 代码需要嵌套在 <% %>中，称为脚本片段。
JSP 文件可以放置在 WEB 应用程序中除了 WEB-INF 及其子目录外的其它任何目录中，JSP 页面的访问路径和普通的 HTML 页面的访问路径形式完全一样。为什么不能放 WEB-INF 下面呢？原因是 WEB-INF 下放到都是些 lib、classes、web.xml 等隐私文件，是不允许被外界直接访问的。但是通过转发可以实现对该目录的访问，后面会介绍。

JSP 的运行原理

我们很好奇，Java 代码和 HTML 页面时怎么融合的啊？
WEB 容器（Servlet 引擎）在接收到以 .jsp 为扩展名的 URL 的访问请求时，它会把该访问请求交给 JSP 引擎去处理。每个 JSP 页面在第一次被访问时，JSP 引擎会将它翻译成一个 Servlet 源程序放到项目的部署目录 \workspace.metadata.plugins\org.eclipse.wst.server.core\tmp0\work\Catalina\localhost\javaweb\org\apache\jsp 中，接着再把这个 Servlet 源程序编译成 .class 文件，然后再由 Servlet 引擎像调用普通 Servlet 程序一样的方式来装载和解释执行。

JSP 隐含对象

什么意思呢？从上面 JSP 引擎为我们翻译的源码中看到，我们嵌套在页面中的代码最终是放在了 public void _jspService(HttpServletRequest request, HttpServletResponse response) 这个方法中，所谓的隐含对象即该方法作用域中提供给我们调用的变量，包括两个参数和 6 个局部变量，实际还有一个叫 exeception 的隐含对象，共 9 个隐含对象，√ 为常用对象：

√ HttpServletRequest request;//
HttpServletResponse response;//在 JSP 页面中一般不会调用该对象的任何方法
√ PageContext pageContext = null;//页面上下文，可以从该对象中获取到其他 8 个隐含对象，也可以获取到当前页面的所有其他信息 （在学习自定义标签的时候会用到它）
√ HttpSession session = null;//代表浏览器和服务器的一次会话
√ ServletContext application = null;//代表当前 WEB 应用
ServletConfig config = null;//代表当前 JSP 对应的 Servlet 的 ServletConfig 对象，几乎不用(这里我们发现，虽然说 JSP 本质上是 Servlet，但是我们在用的时候并不用在 web.xml 文件中配置，只有特殊需要的时候才去配置，很少用)

√ JspWriter out = null;//调用 out.println() 可以直接把字符串打印到浏览器上
Object page = this;//指向当前 JSP 对应的 Servlet 对象的引用，但为 Object 类型，只能调 Object 的方法，开发几乎不用
√ Throwable exeception;//只能在页面声明为错误页面 <%@ page isErrorPage=”true”%>时才可以使用
顺序记忆：pageContext < request < session < application （对属性的作用域的范围从小到大）

JSP 脚本片段

JSP 脚本片段是指嵌套在 <% %>中的一条或多条 Java 代码，多个脚本片段中的代码可以相互访问，在多个脚本片段之间还可以嵌入文本、HTML 标记和其他 JSP 元素等。这个不难理解，因为最终它们都被翻译到 _jspService 方法中。就是有一点，写多了自己就被绕晕了！

脚本片段中的 Java 代码必须严格遵循 Java 语法，以分号结尾

JSP 声明

JSP 声明将 Java 代码置于 <%! %> 中，它里面的代码通过 JSP 引擎翻译后会被放到 Servlet 源程序的 _jspService 方法的外面。所以，JSP 声明可以用于定义 JSP 页面转换成的 Servlet 程序的静态代码块、成员变量、方法等，多个静态代码块、变量、函数可以定义在一个 JSP 声明中，也可以分别单独定义在多个 JSP 声明中。

注意：由于 JSP 隐式对象的作用范围仅限于 _jspService() 方法，所以在 JSP 声明中是不能使用这些隐式对象的，很好理解
说是这么说，真正开发时很少在 JSP 页面中这样用
