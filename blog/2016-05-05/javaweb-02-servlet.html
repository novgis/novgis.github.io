(2) Servlet

  • Java Web

摘要：
Java Servlet 是和平台无关的服务器端组件，它运行在 Servlet 容器中。Servlet 容器负责 Servlet 和客户的通信以及调用 Servlet 的方法。Servlet 和客户的通信采用“请求/响应”的模式。

Servlet 可以完成如下功能：

  • 创建并返回基于客户请求的动态 HTML 页面
  • 创建可嵌入到现有 HTML 页面中的 HTML 片段
  • 与其它服务器资源（如数据库或基于 Java 的应用程序）进行通信

构造器：只被调用一次！只有在第一次请求 Servlet 时，创建 Servlet 的实例，调用构造器。这说明 Servlet 是单实例的！（既然是单实例那就可能存在线程安全问题（要深入去看 Java 是如何实现这个单例的），所以不推荐在 Servlet 中写全局变量。）

关于什么时候被实例化，在配置 Servlet 的时候

init 方法：只被调用一次。在创建好实例后立即被调用，用于初始化当前 Servlet。

为什么不在构造器中初始化而要在 public void init(ServletConfig config) 方法中执行初始化工作，因为该方法传入一个 ServletConfig 类型的参数，这个参数有其特殊作用，是构造器所不及的。

service 方法：被多次调用。每次请求都会调用该方法。用于实际处理请求的方法。

destroy 方法：只被调用一次。在当前 Servlet 所在的 Web 应用被卸载前调用，用于释放当前 Servlet 所占用的资源。
