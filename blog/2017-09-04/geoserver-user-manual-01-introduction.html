一、GeoServer 简介[译]

  • geoserver
gis

本节是对 GeoServer 项目的概述，介绍项目的背景以及 GeoServer 的应用领域。对于想直接上手操作的童孩，请跳过本节，从安装部署章节开始学习。

原文地址

GeoServer 概述

  Geoserver 是一款用 Java 编写的开源地图服务器，允许用户共享和编辑地理空间数据。利用 GeoServer 可以很方便的发布基于开放标准的地理空间数据。

  GeoServer 是由开源社区驱动的项目，其开发、测试和技术支持都是由世界各地的开源贡献者或组织进行维护。

  GeoServer 是对 OGC 发布的 WFS、WCS、WMS等地图协议的参考实现，构建了 Web 地理空间信息技术的核心组件。

GeoServer 的由来

  2001 年一个位于纽约的非营利性孵化机构 TOPP 创建了 GeoServer 项目，旨在构建一套工具来辅助政府公开政务信息。GeoServer 项目的最初设想就是通过对地理空间数据的共享使得公民可以参与政府的一些城市规划等工作。

  GeoServer 的创始人设想了一套类似于万维网的 Web 地理空间信息技术(Geospatial Web)。像万维网人们可以搜索和下载文本信息一样简单，通过 Web 地理空间信息技术，人们可以搜索和下载地理空间数据，而数据的提供者则可以通过这项技术在网络上发布直接供用户访问的地理空间数据，这与以往通过非常繁琐的技术共享地理信息数据完全相反。

  GeoServer的技术人员先前创建了 GeoTools 项目，一套开源的 GIS Java 套件，用于处理 shapfiles、Oracle databases、ARCSDE等各类地理空间数据。

  在 GeoServer 项目创建的同时期，OpenGIS 联盟（现今的 OGC）也正在着手制定 WFS 和 WMS 标准。WFS 协议通过 GML 数据交换格式定义一套协议使得地理空间数据可以在网络上直接访问。WMS 定义了一套由地理空间数据创建的地图的瓦片格式的协议来实现地理空间数据的共享。

  另外一些与 GeoServer 相关的项目，比如 Refractions Research 创建的 PostGIS，提供了一款开源的地理空间数据库； MetaCarta 创建的 OpenLayers，提供了 Web 端的地图可视化框架。这些项目都对发挥 GeoServer 的功能起到很大的促进作用。

  过去十年间，GeoServer 实现了对来自不同数据源的地理空间数据的读取和发布，接下来的十年，GeoServer 将继续有效地推进地理空间数据共享的使命。

参与项目

  GeoServer 因你而精彩！

  GeoServer 完全由开源社区驱动，不分用户和开发者、不分生产者和消费者，有许多种方式参与到 GeoServer 项目的研发当中，任何人都是项目的宝贵的贡献者。

参与开发

  最直接的参与 GeoServer 项目的方式就是参与开发。开发者们一开始可以通过一些 BUG 修复或一些小的补丁程序参与进来，进而在掌握整个系统架构之后有更大的参与力度。而我们的开发人员也非常乐意帮助你来学习这套系统，有很好的代码注释和完善的文档支持。

  项目地址在 GitHub 上，任何人都可以提交 pull requests，通过项目组的评估后您的提交将得到采纳。了解更多关于如何贡献代码，我们建议您订阅 GeoServer 的开发者邮件。下面会提到如何订阅邮件。

文档修正

  另外一个参与项目的重要途径是参与文档修正。无论是新增一些教程或是修正一些错误，都将使得项目更健全。而您不必像上面那样参与开发。

  我们的官方文档也可以通过在 GitHub 上提交 pull requests，通过项目组评审后将合并到主版本中。

邮件订阅

GeoServer 维护着两组邮件：

  • GeoServer 用户
  • GeoServer 开发者

  用户邮件主要是针对在使用 GeoServer 过程中的用户，而开发者邮件更多关注的是开发者的问题，如果有人在这里提问而你知晓答案，烦请为他们提供帮助。

  这些邮件对学习 GeoServer 的新手是一个很好的资源，可以在上面提出问题和参与贡献代码。

IRC

  用户可以加入 GeoServer 的网络即时聊天频道，来在线实时协作。GeoServer 的项目组开发者也偶尔会在频道中回复大家。

BUG 提交

  如果在使用 GeoServer 过程中出现问题，可以通过上面的邮件反馈给我们。GeoServer 使用 JIRA 进行 BUG 跟踪，管理问题。你可能得首先在网站上创建一个账户。

  GeoServer 鼓励所有人通过 JIRA 或者 GItHub 提交问题或解决问题的方案。

翻译文档

  参与 GeoServer 文档的翻译。

优化建议

  可以通过邮件或者 GiHub 提出您的优化建议。

宣传

  加大 GeoServer 的宣传力度。

赞助

  对 GeoServer 提供赞助。
